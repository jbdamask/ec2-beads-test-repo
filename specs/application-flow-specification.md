## 10. State Machine Definitions

### 10.1 Weather Data State Machine

```typescript
type WeatherDataState =
  | { status: 'idle' }
  | { status: 'loading'; location: Location }
  | { status: 'success'; data: WeatherData; fetchedAt: Date }
  | { status: 'error'; error: Error; lastValidData?: WeatherData }
  | { status: 'stale'; data: WeatherData; fetchedAt: Date }
  | { status: 'refreshing'; data: WeatherData; fetchedAt: Date };

// State transitions
// idle â†’ loading (on location change)
// loading â†’ success (on data received)
// loading â†’ error (on fetch failure)
// success â†’ stale (after 5 minutes)
// stale â†’ refreshing (on refresh triggered)
// refreshing â†’ success (on data received)
// refreshing â†’ stale (on refresh failure, keep old data)
// error â†’ loading (on retry)
```

### 10.2 Location Search State Machine

```typescript
type SearchState =
  | { status: 'idle' }
  | { status: 'typing'; query: string }
  | { status: 'debouncing'; query: string }
  | { status: 'searching'; query: string }
  | { status: 'results'; query: string; results: Location[] }
  | { status: 'error'; query: string; error: Error }
  | { status: 'selected'; location: Location };

// Transitions
// idle â†’ typing (on input focus + keypress)
// typing â†’ debouncing (on keypress, reset 300ms timer)
// debouncing â†’ searching (after 300ms)
// searching â†’ results (on API response)
// searching â†’ error (on API failure)
// results â†’ selected (on result click)
// selected â†’ idle (on blur/navigation)
// any â†’ idle (on escape/clear)
```
